# Story 09-09: 自动回复逻辑门控

> Epic: AI 多人群聊 (Group Chat)
> Priority: Medium
> Points: 2

## 概述
实现群聊"主动权"的精细化控制。通过同步前端设置与后端调度引擎，允许用户一键切换群聊模式：是让 AI 们在群里大显身手（自动回复），还是保持安静，仅在被呼叫时（@）才响应。

## 用户故事

### US-1: 宁静模式
**作为** 用户，
**我希望** 能在群设置里关闭"自动回复"，
**以便** 只有在我精准指名（@）某个 AI 时，它才出来说话，减少不必要的干扰。

### US-2: 动态社交模式
**作为** 用户，
**我希望** 开启"自动回复"后，AI 们能根据我们的聊天内容自发活跃群气氛，
**以便** 感知到一个真实的社交沙盒。

## 功能范围

### 包含
- **设置 API 扩展**: 为群组设置接口增加 `auto_reply` 字段的读写支持。
- **调度拦截器逻辑**: 在后端 `process_group_message` 流程中，根据开关状态决定是否进入 Hidden Admin 的决策环节。
- **状态一致性**: 确保多端登录（如 Electron 和 Web）看到的群开关状态是同步的。

### 不包含（后续迭代）
- 定时自动开启/关闭自动回复
- 基于敏感词或特定话题的条件式自动回复

## 验收标准

- [ ] AC-1: 群详情页面的"自动回复"开关操作流畅，并能实时保存至数据库。
- [ ] AC-2: 开关关闭时，普通消息不触发任何后端生成任务。
- [ ] AC-3: 开关关闭时，依然保留 @ 触发的最高优先级，确保交互不断层。
- [ ] AC-4: UI 上通过文案或 Toast 明确提示用户当前的回复模式。
- [ ] AC-5: 验证开关状态对历史会话无影响，仅作用于后续消息。
